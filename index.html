Laporan 1 - Pengenalan Mikrotik

Mikrotik Winbox


Pengertian Mikrotik
MikroTik adalah perusahaan yang bergerak di bidang teknologi jaringan, terutama dikenal dengan perangkat keras dan perangkat lunak jaringan yang digunakan untuk keperluan routing dan manajemen jaringan. MikroTik didirikan di Latvia pada tahun 1996, dan produknya meliputi router, switch, access point, serta perangkat lunak RouterOS.
Jenis - Jenis Mikrotik
Router
Produk-produk seperti hEX Series, CCR Series, dan hAP Series adalah bagian dari kategori router. Router MikroTik digunakan untuk mengelola lalu lintas jaringan, baik di rumah, kantor kecil, maupun di jaringan skala besar.

Switch
CRS Series (Cloud Router Switch) dan CSS Series masuk dalam kategori switch. Switch ini digunakan untuk distribusi jaringan dalam jaringan lokal (LAN) dan menawarkan kemampuan switching dengan kecepatan tinggi.

Acces Point
wAP Series dan hAP Series termasuk kategori access point, menyediakan konektivitas Wi-Fi untuk perangkat pengguna di rumah atau kantor.

Wireless System
Produk seperti LHG Series dan SXT Series adalah perangkat nirkabel untuk koneksi point-to-point atau point-to-multipoint, biasanya digunakan untuk jaringan outdoor dan long-range wireless.

Modem LTE/4G
Produk seperti LtAP Series menyediakan koneksi jaringan berbasis LTE untuk wilayah terpencil atau aplikasi mobile, sering digunakan di luar ruangan atau lokasi yang sulit dijangkau.
Fungsi / Penerapan penggunaan Mikrotik
VPN (Virtual Private Network): Menghubungkan jaringan secara aman melalui internet.
Hotspot Gateway: Mengelola akses internet untuk pengguna dengan fitur login dan pembatasan akses.
Routing: Mengelola lalu lintas data antar jaringan dan perangkat.
Bandwidth Management (QoS): Mengatur dan membatasi penggunaan bandwidth untuk mencegah kemacetan.
Firewall: Mengamankan jaringan dengan memfilter paket data dan mengontrol akses
Kelebihan dan Kekurangan Mikrotik
Kelebihan
- Fitur Lengkap, Menyediakan beragam fitur jaringan seperti routing, firewall, VPN, dan QoS.
- Harga yang lebih Terjangkau Dibandingkan perangkat serupa, dengan menawarkan fitur yang canggih
- Konfigurasi Fleksibel yang Sangat bisa dikustomisasi sesuai kebutuhan jaringan.

Kekurangan
- Kurva Pembelajaran Curam, yang Memerlukan pemahaman teknis yang cukup untuk konfigurasi awal.
- Antarmuka yang Kompleks, Bagi pemula antarmuka bisa terlihat rumit dan membingungkan.
- Layanan dukungan resminya terbatas, banyak pengguna mengandalkan komunitas untuk solusi.
Tipe Lab Mikrotik
Beberapa tipe nya antara lain :
MTCNA Lab: Lab dasar untuk konfigurasi jaringan, routing, dan firewall.
MTCRE Lab: Lab lanjutan fokus pada routing seperti OSPF dan BGP.

Konfigurasi DHCP Server: Menyediakan alamat IP secara otomatis kepada perangkat dalam jaringan.

Winbox adalah aplikasi GUI (Graphical User Interface) yang dikembangkan oleh MikroTik untuk memudahkan manajemen dan konfigurasi perangkat MikroTik. Dengan menggunakan Winbox, pengguna dapat melakukan berbagai pengaturan pada RouterOS secara visual tanpa perlu menggunakan perintah baris (CLI).

Manajemen Wireless: Memungkinkan pengguna untuk mengonfigurasi pengaturan jaringan nirkabel, termasuk SSID, mode operasi, dan keamanan.

Laporan 2 - Konfigurasi IP Address Perangkat Mikrotik

Mikrotik Winbox



PEMBAHASAN

Pengertian IP Address
IP Address (Internet Protocol Address) adalah sebuah alamat unik yang digunakan untuk mengidentifikasi perangkat dalam jaringan komputer, baik jaringan lokal maupun internet.
Jenis - Jenis IP Address
IP Address Publik (Public IP Address)
adalah IP yang digunakan untuk mengidentifikasi perangkat di jaringan internet global dan IP ini unik dan dialokasikan oleh penyedia layanan internet (ISP). contohnya :
  • 192.0.2.1
  • 203.0.113.5
  • 8.8.8.8 (Google DNS Server)

    • IP Publik biasanya unik dan digunakan untuk perangkat yang terhubung langsung ke internet seperti server web, router yang menyediakan akses internet, dll.

    IP Address Privat (Private IP Address)
    Adalah IP yang digunakan di dalam jaringan lokal (LAN) dan tidak dapat diakses langsung dari internet, dan Rentang IP privat sudah ditentukan dan tidak dapat digunakan untuk jaringan publik. Contohnya :
  • Rentang IP Privat Kelas A (10.0.0.0 â€“ 10.255.255.255)
  • Rentang IP Privat Kelas B (172.16.0.0 â€“ 172.31.255.255)
  • Rentang IP Privat Kelas C (192.168.0.0 â€“ 192.168.255.255)

    • IP Privat ini digunakan oleh perangkat seperti komputer, router, atau smartphone yang terhubung ke jaringan lokal (LAN).
    IP Address
    IPv4 (Internet Protocol version 4)
    Alamat IP dengan panjang 32 bit, biasanya ditulis dalam format desimal dipisahkan oleh titik ".", dan IPv4 mendukung sekitar 4,3 miliar alamat, yang mulai habis karena semakin banyak perangkat yang terhubung ke internet.
    Contohnya 192.168.1.1

    IPv6 (Internet Protocol version 6)
    Alamat IP dengan panjang 128 bit, ditulis dalam format heksadesimal dipisahkan oleh tanda titik dua ":", IPv6 diciptakan untuk mengatasi keterbatasan IPv4 dan mendukung jumlah alamat yang jauh lebih besar.
    Contohnya 2001:db8::ff00:42:8329
    Langkah - Langkah Konfigurasi IP Address Perangkat Mikrotik Menggunakan Winbox
  • Pertama hidupkan terlebih dahulu pernagkat mikrotiknya dengan menghubungkan kabel powernya
  • Lalu pasang kabel LAN (RJ45) ke port Ether1 dan ujung satunya lagi ke sumber internet (ISP)
  • Pasang Juga kabel LAN ke port Ether2 dan ujung satunya lagi ke perangkat yang kita gunakan (PC Client)

    • Login ke Mikrotik melalui Winbox
  • Buka aplikasi Winbox, lalu Pada tab Neighbors, pilih perangkat Mikrotik Anda
  • Masukkan Username (default: admin) dan Password (default: kosong/tidak ada)
  • Klik Connect untuk masuk ke antarmuka Mikrotik.

    • Konfigurasi IP Address Menggunakan DHCP Client
  • buka IP > DHCP Client di tab sebelah kiri, Klik tombol "+"
  • Pada kolom Interface, pilih interface WAN (misalnya, ether1).
  • Centang opsi Add Default Route agar rute otomatis ditambahkan Klik Apply dan OK

    • Konfigurasi Gateway (Default Route)
  • Buka IP > Routes.
  • Jika menggunakan DHCP Client, default route akan secara otomatis ditambahkan. Jika tidak Klik tombol "+"Pada kolom Dst. Address, isikan 0.0.0.0/0 (untuk mengarahkan semua lalu lintas keluar).
  • Pada kolom Gateway, masukkan alamat IP gateway yang diberikan oleh ISP (misalnya: 192.168.1.1).
  • Klik Apply dan OK

    • Konfigurasi DNS
  • Buka IP > DNS.
  • Pada kolom Servers, masukkan DNS server yang diberikan oleh ISP atau gunakan DNS umum seperti 8.8.8.8 (Google DNS) dan 1.1.1.1 (Cloudflare DNS).
  • Centang Allow Remote Requests jika Anda ingin perangkat lain di jaringan bisa menggunakan Mikrotik sebagai server DNS.
  • Klik Apply dan OK

    • Konfigurasi NAT (Network Address Translation)
  • Buka IP > Firewall, kemudian pilih tab NAT dan Klik tombol "+"
  • Pada tab General, atur Chain: srcnat, Out. Interface: Pilih interface WAN yang terhubung ke ISP (misalnya, ether1).
  • Pindah ke tab Action, dan pilih Action: masquerade.
  • Klik Apply dan OK

    • Konfigurasi LAN
  • Buka IP > Addresses, dan tambahkan IP untuk jaringan internal (misalnya, 192.168.88.1/24 pada interface ether2).
  • Jika Anda membutuhkan DHCP Server untuk jaringan LAN : Buka IP > DHCP Server, Buat konfigurasi DHCP Server untuk interface LAN (ether2), dan atur range IP yang diberikan ke perangkat di jaringan.

    • Dengan mengikuti langkah-langkah ini, perangkat Mikrotik Anda akan berhasil terhubung ke ISP dan mendistribusikan koneksi internet ke jaringan lokal.

  • Wireless Configuration : Mengkonfigurasi router untuk berfungsi sebagai access point atau client nirkabel

    • Kesimpulan
    Konektivitas Terjamin :
    Konektivitas Terjamin: Setelah semua IP address dan routing statis dikonfigurasi, setiap router dapat berkomunikasi dengan jaringan lain, memastikan konektivitas end-to-end di seluruh jaringan.

    Penggunaan Routing Statis :
    Static routing memberikan kontrol penuh kepada administrator dalam menentukan jalur jaringan, namun membutuhkan konfigurasi manual setiap kali ada perubahan.

    Topologi Terstruktur:
    Dengan topologi yang jelas dan terhubung melalui tiga router, setiap jaringan lokal (LAN) dapat berkomunikasi tanpa hambatan melalui jalur yang telah ditentukan.

    Pentingnya Konfigurasi yang Tepat:
    Kesalahan dalam pengaturan IP atau routing dapat menyebabkan gangguan komunikasi. Oleh karena itu, pengecekan IP, rute, dan status interface sangat penting selama proses konfigurasi.

    Manfaat Static Routing:
    Routing statis cocok untuk jaringan kecil hingga menengah karena menawarkan stabilitas dan menghindari overhead seperti yang terjadi pada dynamic routing, meskipun memerlukan lebih banyak waktu untuk pemeliharaan.

    Laporan 6 - NAT bridge dan NAT firewall

    NAT Bridge dan Firewall



    NAT (Network Address Translation)
    NAT adalah teknik yang digunakan untuk menerjemahkan alamat IP dalam sebuah jaringan. Biasanya digunakan untuk menyembunyikan alamat IP privat di belakang satu atau beberapa alamat IP publik. NAT membantu dalam mengurangi konsumsi alamat IP publik dan meningkatkan keamanan jaringan
    NAT Pada Menu Firewall
    Pada router atau perangkat jaringan, NAT umumnya dikonfigurasi melalui Firewall karena di situlah proses filtering dan manajemen alur paket terjadi
    Ciri dan Fungsi NAT di Menu Firewall :
  • Pengaturan Routing dan Translasi : NAT di sini berperan dalam mengatur bagaimana IP privat diterjemahkan ke IP publik saat paket melewati firewall
  • Source NAT (Masquerade) : Digunakan untuk menyembunyikan IP privat di balik satu IP publik, terutama untuk koneksi keluar (outbound)
  • Destination NAT (DNAT) : Memungkinkan paket dari luar dialihkan ke alamat internal tertentu (misalnya server lokal)
  • Control dan Filter Paket : Mengatur port forwarding dan QoS
  • Keamanan : Berperan dalam memperkuat keamanan jaringan dengan menyembunyikan struktur internal
    • Kelebihan NAT di Firewall :
  • Fleksibilitas dan kontrol lebih baik terhadap traffic
  • Mudah digunakan untuk SNAT dan DNAT
  • Mendukung konfigurasi kompleks seperti VPN dan tunneling
    • Contoh Penggunaan NAT di Firewall :
  • Mengarahkan trafik dari internet ke server internal menggunakan port forwarding
  • Koneksi internet di mana banyak perangkat privat menggunakan satu alamat IP publik
    • NAT Pada Menu Bridge
    Bridge bekerja pada lapisan 2 OSI (Data Link Layer), sementara NAT terkait dengan lapisan 3 (Network Layer). Karena itu, NAT di menu Bridge memiliki tujuan dan mekanisme berbeda dibanding NAT di Firewall.
    Ciri dan Fungsi NAT di Menu Bridge :
  • Tidak memiliki kontrol trafik selengkap NAT di firewall
  • Jarang digunakan, biasanya untuk segmentasi khusus
  • Digunakan dalam bridging dengan translasi IP minimal, seperti komunikasi antar VLAN
    • Keterbatasan NAT di Bridge :
  • Tidak cocok untuk jaringan skala besar
  • Kurang fleksibel dibanding NAT di firewall
    • Kesimpulan NAT di Firewall vs Bridge
    NAT di Firewall : Lebih cocok untuk routing dan pengamanan jaringan karena menawarkan fleksibilitas dan kontrol tinggi
    NAT di Bridge : Digunakan dalam skenario layer 2 tertentu tetapi kurang populer
    AP Bridge vs Bridge
    AP Bridge
    AP Bridge adalah mode di mana Access Point (AP) memungkinkan perangkat klien terhubung secara wireless dan menghubungkan beberapa perangkat jaringan melalui akses nirkabel. Fungsi utamanya adalah menghubungkan jaringan kabel dan nirkabel menjadi satu kesatuan.
    Ciri dan Fungsi AP Bridge
  • Meneruskan frame wireless ke perangkat kabel
  • Memancarkan SSID untuk diakses oleh klien
  • Mendukung banyak klien wireless
  • Menghubungkan jaringan wireless dan kabel
    • Kapan AP Bridge Digunakan?
  • Di kantor atau rumah dengan perangkat wireless dan kabel
  • Saat menghubungkan jaringan kabel dan wireless dalam satu subnet
    • Bridge
    Bridge adalah mekanisme yang menghubungkan dua atau lebih jaringan di lapisan 2. Fungsinya adalah meneruskan frame Ethernet antar segmen jaringan tanpa mengubah alamat IP atau protokol.
    Ciri dan Fungsi Bridge :
  • Menghubungkan VLAN dalam satu jaringan
  • Tidak memancarkan SSID
  • Segmentasi jaringan : Memisahkan lalu lintas untuk mencegah broadcast storm
  • Layer 2 forwarding : Meneruskan frame tanpa melihat paket layer 3
    • Kapan Bridge Digunakan ?
  • Untuk segmentasi yang mencegah broadcast berlebihan
  • Dalam jaringan LAN untuk menghubungkan dua segmen fisik
    • Kesimpulan AP Bridge vs Bridge
    AP Bridge : Menghubungkan perangkat wireless dengan jaringan kabel melalui Access Point
    Bridge : Berfungsi menghubungkan beberapa segmen jaringan di layer 2 tanpa memancarkan sinyal wireless
    Rangkuman Akhir
  • NAT di Firewall lebih berguna untuk routing dan manajemen paket secara kompleks, sementara NAT di Bridge hanya digunakan dalam skenario layer 2 tertentu
  • AP Bridge memungkinkan integrasi wireless dan kabel melalui Access Point, sedangkan Bridge bekerja di lapisan 2 untuk menghubungkan segmen jaringan kabel.

    GitHub
    GitHub adalah platform berbasis web yang digunakan untuk menyimpan dan mengelola kode sumber serta proyek pengembangan perangkat lunak. Platform ini menggunakan Git, sistem kontrol versi yang memungkinkan pengembang untuk melacak perubahan dalam kode dan berkolaborasi dengan orang lain dalam pengembangan perangkat lunak.
    Fungsi GitHub
    Sebagai Repository : Yaitu Tempat untuk menyimpan proyek atau kode sumber
    Branching dan Merging : Memungkinkan pengembang menggabungkan perubahan ke kode utama.
    Pull Request : Mengusulkan perubahan untuk ditinjau, dikomentari, dan digabungkan ke proyek utama.
    Issues dan Discussions: Untuk melaporkan masalah, berdiskusi, atau memberikan masukan terkait proyek
    Actions : Fitur untuk mengotomatiskan alur kerja, seperti CI/CD (Continuous Integration/Continuous Deployment)
    Keunggulan GitHub
    Kolaborasi Mudah : Memungkinkan kerja sama tim melalui fitur seperti pull request dan code review
    Kontrol Versi : Melacak perubahan kode secara efektif dengan Git
    Integrasi CI/CD : Mendukung otomatisasi alur kerja pengembangan
    Manajemen Proyek : Fitur seperti Issues dan Projects untuk pengorganisasian tugas
    Aksesibilitas dan Komunitas Besar : Banyak proyek open source dan dokumentasi tersedia untuk pembelajaran
    Register GitHub
  • Hal pertama yang dilakukan untuk membuat akun GitHub yaitu search GitHub di browser, atau bisa juga klik link ini https://github.com/
  • Kemudian akan di arahkan ke laman githubnya, lalu klik sign up untuk membuat akun githubnya.
  • Lalu masukkan email Anda, pastikan akun email yang dimasukkan aktif karena nanti akan dimintai kode verifikasi yang akan dikirim melalui email tersebut.
  • Setelah memasukkan email, langkah selanjutnya yaitu membuat password, pastikan password yang dimasukkan itu bersifat sulit, berikan kombinasi simbol, angka, huruf besar, dan huruf kecil agar memastikan password tersebut sangat sulit untuk diakses oleh orang lain. Jangan lupa juga untuk mencatat email dan password akun tersebut agar tidak lupa, setelah itu klik continue.
  • Jika akun telah selesai dibuat, maka tahapan selanjutnya yaitu masuk menggunakan akun yang baru saja dibuat, nanti akan diminta kode verifikasi lewat email Anda.
  • Selamat Akun GitHub pun telah berhasil dibuat.
